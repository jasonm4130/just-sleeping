# They're Just Sleeping — agent guide

A static Astro photo-zine (deadpan British humour, paper-collage aesthetic) deployed
to **Cloudflare Workers static assets**. Live: https://theyrejustsleeping.com

`README.md` has the full "what & why". This file is the working guide for changing it.

## Commands

- `pnpm dev` — dev server at http://localhost:4321
- `pnpm build` — `astro check` (typecheck) + static build to `dist/`
- `pnpm preview` — serve the built `dist/` locally
- `pnpm lint` / `pnpm format` — Biome
- Content regen (needs `MOONSHOT_API_KEY`): `node scripts/shrink.ts` → `node scripts/caption.ts` → `node scripts/pick-featured.ts`

## Deploy — push to `main`

**Cloudflare auto-deploys on push to `main`** (it runs `pnpm build` and serves `dist/`).
Do **not** run local deploys. Just commit and push. The `pnpm deploy` script
(`wrangler deploy`) is a manual fallback only — the canonical path is git push.

## Architecture

- `src/pages/` — `index.astro` (single page; composes the components) + `404.astro`
- `src/layouts/Layout.astro` — `<head>`, font preloads, theme-before-paint inline script
- `src/components/` — Hero · Faq · FeaturedGrid→SpecimenCard · ArchiveGrid · Footer · NightToggle. Each owns a scoped `<style>`.
- `src/styles/` — `tokens.css` (design tokens) · `fonts.css` (`@font-face`) · `global.css` (`@layer` reset/base/utilities, the six `tjs*` keyframes, `.u-grain`)
- `src/data/specimens.ts` — curated featured set (generated by `pick-featured.ts`)
- `scripts/captions.json` — per-photo AI copy (generated by `caption.ts`)
- `src/photos/archive/` (214 source photos) · `src/photos/featured/` (the collection) — optimized at build by `astro:assets`

## Conventions

- **CSS, no framework.** Two-tier tokens: primitives → semantic aliases in `tokens.css`. Components reference ONLY semantic tokens (`--color-*`, `--font-*`). Night mode re-maps the semantic tier via `:root[data-theme="night"]` — no JS re-render.
- **Scoped styles.** Each component's `<style>` is scoped; anything shared (`tjs*` keyframes, `.u-grain`, `.u-container`, `.sr-only`) lives in `global.css`.
- **Images via `astro:assets` only** (`<Image>`/`<Picture>`), never raw `<img>` to a source photo. Build-time only — no runtime/CF Images, no per-request cost.
- **Minimal JS.** Only `poke.ts` (counter) and the night toggle ship JS, persisted to `localStorage`.
- **TS strict** (`astro/tsconfigs/strict`); `pnpm build` runs `astro check` — keep it green.
- LSP-first navigation (see `~/Work/Git/CLAUDE.md`).

## Performance & a11y invariants — don't regress

Targets Lighthouse **100** on Performance / Accessibility / Best-Practices / SEO, mobile + desktop. Keep:

- **Fonts self-hosted & glyph-subset** — `public/fonts/*.woff2`, `@font-face` in `fonts.css`, subset to the glyphs the site actually renders (ASCII + `· — ' ' " " …`). No Google Fonts `<link>` (it was render-blocking, ~1.75s on mobile). The two LCP-critical fonts are `<link rel="preload">`ed in `Layout.astro`. **Gotcha: if you add copy with new glyphs (accents, new symbols), re-subset the woff2** (`pyftsubset --unicodes=…`, see the `fonts.css` header) and widen its `unicode-range` to match — otherwise the new glyph silently falls back to a system font.
- **Every photo renders grayscale** (`filter: grayscale(1) contrast(...)`), so image `quality` is deliberately low (40–55) — fewer bytes, zero visible change. Keep it low.
- **Hero `gary` is the LCP image** → `loading="eager" fetchpriority="high"`. Every other image is `loading="lazy"`.
- **CSS is inlined** (`build.inlineStylesheets: "always"` in `astro.config.mjs`) — no render-blocking stylesheet request.
- **FAQ `<dl>`** must contain only `<div>` groups each wrapping `<dt>`+`<dd>`. The "Q." marker is a CSS `::before`, not an element — a stray child fails the `definition-list` audit.
- `_headers` long-caches `/_astro/*` and `/fonts/*` as `immutable`.

## Verify before pushing

1. `pnpm build` must pass (astro check).
2. `pnpm preview`, then Lighthouse against `http://localhost:4321`. Note local preview is HTTP/1.1 with no compression, so it **under-reports** vs the Cloudflare edge — the authoritative check is Lighthouse on the live URL after deploy.
